MCP Server

Rule MCP Server

Connect Claude and other AI clients to Rule — the marketing automation platform for email & SMS.

1 Prerequisites

RequirementDetails
Node.js Version 18 or later — nodejs.org
Rule API Key Found in your Rule account under Settings → API
MCP-compatible client Claude Desktop, Claude Code, Cursor, or any MCP client

2 Install the server

Clone the repository and install dependencies:

git clone https://github.com/your-org/rule-mcp-server.git
cd rule-mcp-server
npm install

Note the absolute path to index.js — you will need it in the next step:

pwd
# e.g. /Users/you/rule-mcp-server

3 Configure your MCP client

Select your client below to see the matching configuration:

Claude Desktop

macOS / Windows app

Claude Code

CLI / VS Code extension

Cursor

AI-native code editor

Open (or create) ~/Library/Application Support/Claude/claude_desktop_config.json and add:

{
  "mcpServers": {
    "rule": {
      "command": "node",
      "args": ["/absolute/path/to/rule-mcp-server/index.js"],
      "env": {
        "RULE_API_KEY": "your-rule-api-key-here"
      }
    }
  }
}

Restart Claude Desktop after saving. The Rule tools will appear in the tool picker.

Run the following command to add the server to your Claude Code config:

claude mcp add rule \
  --command node \
  --args "/absolute/path/to/rule-mcp-server/index.js" \
  --env RULE_API_KEY=your-rule-api-key-here

Or edit ~/.claude/settings.json manually using the same JSON structure as the Claude Desktop config above.

Open Cursor Settings → MCP and add a new server entry:

{
  "mcpServers": {
    "rule": {
      "command": "node",
      "args": ["/absolute/path/to/rule-mcp-server/index.js"],
      "env": {
        "RULE_API_KEY": "your-rule-api-key-here"
      }
    }
  }
}

4 Set your API key

Replace your-rule-api-key-here in the config above with your actual Rule API key.

⚠

Never commit your API key to version control. Use environment variables or a secrets manager instead.

Alternatively, export the key in your shell and reference it from the config:

export RULE_API_KEY="your-rule-api-key-here"

5 Verify the installation

Start the server manually to confirm it runs without errors:

RULE_API_KEY=your-rule-api-key-here node index.js

You should see no output — the server waits for MCP messages over stdio. Press Ctrl+C to exit.

Inside your AI client, ask something like:

List my Rule subscribers

Troubleshooting

ProblemSolution
Server not found Check that the path in args is absolute and the file exists
401 Unauthorized Verify your RULE_API_KEY is correct and active
Node not found Ensure Node.js 18+ is installed and node is on your PATH
Tools not appearing Restart your MCP client after updating the config file
